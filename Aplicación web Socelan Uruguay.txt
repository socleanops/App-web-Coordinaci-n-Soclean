🎯 DESCRIPCIÓN GENERAL DEL PROYECTO
Desarrolla una aplicación web empresarial completa llamada "CoreOps" (o el nombre que defina el cliente), diseñada para gestionar de forma integral los servicios que brinda una empresa, el control de personal, facturación a clientes y nómina. La app debe ser modular, escalable, intuitiva y visualmente atractiva.

🛠️ STACK TECNOLÓGICO

Frontend: React 18 + TypeScript + Vite
Estilos: Tailwind CSS + shadcn/ui (componentes accesibles y elegantes)
Base de datos: Supabase (PostgreSQL + Auth + Storage + Realtime)
Estado global: Zustand o React Query (TanStack Query)
Formularios: React Hook Form + Zod (validación)
Tablas y filtros: TanStack Table v8
Gráficas: Recharts
Fechas: date-fns
Exportación: xlsx + jsPDF
Iconos: Lucide React


🎨 DISEÑO Y UX — ESPECIFICACIONES VISUALES
Estética general

Tema: Profesional-moderno con modo claro y oscuro
Paleta de colores principal:

Primario: #1E40AF (azul profundo)
Acento: #0EA5E9 (azul cielo)
Éxito: #10B981 (verde esmeralda)
Alerta: #F59E0B (ámbar)
Peligro: #EF4444 (rojo)
Fondo claro: #F8FAFC
Fondo oscuro: #0F172A


Tipografía:

Títulos: Sora o Plus Jakarta Sans (Google Fonts)
Cuerpo: DM Sans


Bordes: redondeados (rounded-xl), con sombras suaves (shadow-md)
Sidebar: colapsable, con íconos y labels, efecto glassmorphism leve
Animaciones: transiciones suaves en navegación, skeleton loaders, toast notifications
Responsive: 100% adaptable a escritorio, tablet y móvil

Tablas con paginación, búsqueda en tiempo real y filtros avanzados
Formularios con validación inline antes de enviar
Confirmación antes de eliminar registros
Dashboard con KPIs y gráficas visibles al primer vistazo


📦 MÓDULOS DE LA APLICACIÓN

1. 🔐 AUTENTICACIÓN Y USUARIOS

Login con email y contraseña (Supabase Auth)
Recuperación de contraseña por email
Roles de usuario:

superadmin: acceso total
admin: gestión operativa completa
supervisor: ver y aprobar asistencia y servicios
facturador: acceso solo a facturación y nómina
funcionario: solo ver su perfil, horario y asistencia




2. 👥 MÓDULO DE FUNCIONARIOS (RRHH)
Gestión de empleados:

Crear, editar y desactivar funcionarios
Campos: nombre completo, cédula/DNI, cargo, departamento, fecha de ingreso, tipo de contrato, salario base, datos de contacto, foto de perfil
Estado: activo / inactivo / vacaciones / licencia

Documentos:

Subir y visualizar documentos por funcionario (contratos, certificados, etc.)
Alertas de documentos próximos a vencer


3. 📅 MÓDULO DE HORARIOS

Crear turnos/horarios semanales por funcionario o por grupo
Vista de calendario semanal y mensual
Asignar funcionarios a turnos específicos
Gestionar excepciones (días libre, permisos, cambios de turno)
Exportar horario en PDF o Excel
Vista de "Mi Horario" para cada funcionario


4. ✅ MÓDULO DE ASISTENCIA

Registro de entrada y salida (manual por admin o via código/pin por funcionario)
Visualización diaria, semanal y mensual de asistencia
Indicadores: presente, ausente, tardanza, salida anticipada
Filtros por fecha, funcionario, departamento
Exportar reporte de asistencia en Excel/PDF
Alertas automáticas por ausencias repetidas
Cálculo automático de horas trabajadas, horas extra y horas faltantes


5. 🗂️ MÓDULO DE SERVICIOS

Catálogo de servicios que ofrece la empresa (crear, editar, activar/desactivar)
Campos: nombre, descripción, categoría, precio base, duración estimada, notas
Asignación de servicios a clientes
Estado del servicio: pendiente / en curso / completado / cancelado
Historial de servicios por cliente
Filtros por fecha, estado, cliente, tipo de servicio
Asignar funcionarios responsables a cada servicio


6. 👔 MÓDULO DE CLIENTES

Crear y gestionar clientes (personas naturales o empresas)
Campos: nombre/razón social, RUT/NIT, contacto, dirección, condición de pago
Historial de servicios prestados
Balance de cuenta (servicios facturados vs. pagados)
Búsqueda y filtros avanzados


7. 🧾 MÓDULO DE FACTURACIÓN

Generar facturas a partir de servicios prestados
Campos: cliente, servicios incluidos, fecha, número de factura (automático), condiciones de pago, impuestos (configurable), descuentos
Estados de factura: borrador / emitida / pagada / vencida / anulada
Historial y listado de facturas con filtros por cliente, fecha, estado, monto
Vista previa de factura y exportación a PDF
Envío de factura por email (integración con Resend o SMTP)
Dashboard de facturación: ingresos del mes, facturas vencidas, por cobrar


8. 💰 MÓDULO DE NÓMINA

Generación de nómina mensual/quincenal
Cálculo automático basado en:

Salario base del funcionario
Horas trabajadas (desde módulo de asistencia)
Horas extra (con recargo configurable)
Descuentos por ausencias/tardanzas
Deducciones fijas (seguridad social, impuestos — configurables)
Bonificaciones adicionales (campo manual)


Revisión y aprobación de nómina antes de cerrar
Exportar nómina en Excel (formato estándar para contabilidad)
Exportar recibo de pago individual en PDF por funcionario
Historial de nóminas cerradas


9. 📊 DASHBOARD PRINCIPAL
Panel de control con:

Total de funcionarios activos
Servicios en curso hoy
Facturas pendientes de cobro (monto total)
Asistencia del día (presente/ausente en %)
Gráfica de ingresos por mes (barras)
Gráfica de servicios por categoría (dona/pie)
Próximos vencimientos de facturas (tabla)
Actividad reciente (timeline de últimas acciones)


10. ⚙️ CONFIGURACIÓN DEL SISTEMA

Datos de la empresa (nombre, logo, dirección, RUT, datos fiscales)
Configuración de impuestos (IVA u otros)
Tipos de contrato disponibles
Departamentos y cargos de la empresa
Categorías de servicios
Configuración de recargos por horas extra
Plantilla personalizable de factura (logo, colores, pie de página)
Gestión de usuarios y roles

🔒 SEGURIDAD Y REGLAS DE ACCESO (Row Level Security en Supabase)

Cada tabla con RLS activado
Los funcionarios solo ven sus propios registros de asistencia y horario
Solo admins y superadmins pueden crear/editar/eliminar registros sensibles
Facturadores tienen acceso de solo lectura a datos de nómina
Logs de auditoría para cambios críticos (edición de salarios, cierre de nómina, emisión de facturas)


🚀 INSTRUCCIONES DE DESARROLLO PARA LA IA

Comienza por: configuración del proyecto, autenticación y layout base con sidebar
Sigue con: módulo de funcionarios (es el núcleo de los demás módulos)
Luego: horarios y asistencia (dependen de funcionarios)
Después: servicios, clientes y facturación (relacionados entre sí)
Finalmente: nómina (consume datos de asistencia y funcionarios) y dashboard (consume todo)
Cada módulo debe quedar funcional antes de pasar al siguiente
Usa datos de ejemplo (seed data) para poblar la base de datos desde el inicio
Implementa filtros y exportación en todas las tablas principales
El sistema debe ser multi-idioma ready (textos en constantes, no hardcodeados)
Agrega comentarios en el código para facilitar futuras modificaciones


✅ CRITERIOS DE CALIDAD

 Todas las vistas son responsive (mobile-first)
 Skeleton loaders en carga de datos
 Manejo de errores con mensajes amigables al usuario
 Confirmación antes de acciones destructivas
 Exportación disponible en tablas principales
 Dashboard carga en menos de 2 segundos
 Código TypeScript estricto, sin any
 Componentes reutilizables y bien nombrados
