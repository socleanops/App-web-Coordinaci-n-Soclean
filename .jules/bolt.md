## 2024-05-18 - Caching Optimization and Strict Typing
**Learning:** Over-aggressive cache clearing (`localStorage.clear()`) combined with Supabase auth state changes can trigger infinite re-renders or application freezes by accidentally destroying React Query or Vite internal states. Additionally, using `any` indiscriminately masks type issues and creates fragile components that are difficult to refactor safely. `catch (error: Record<string, unknown>)` is syntactically invalid TypeScript; catch variables must be strictly `any` or `unknown`.
**Action:** Replace `localStorage.clear()` with targeted `localStorage.removeItem('specific-key')`. Add aggressive `Cache-Control` headers for static assets in `vercel.json` to prevent unnecessary network requests. Refactor `any` to `unknown` or specific types throughout the codebase, especially in `catch` blocks and component props, ensuring adherence to strict TypeScript rules without breaking the build.
